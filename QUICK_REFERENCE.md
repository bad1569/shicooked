# Quick Reference - Jobs Management System

## What Was implemented

### ✅ Dynamic Jobs Data
- Jobs are no longer hardcoded
- All job data stored in Firebase Realtime Database
- Real-time updates across all pages

### ✅ Admin Capabilities
- Add new jobs
- Edit existing jobs
- Delete jobs
- View all jobs in a table

### ✅ Homepage Display
The homepage now displays jobs in 4 categories (each showing 4 jobs):

| Category | Filter Type | Description |
|----------|------------|-------------|
| Recent Jobs | "all" | Latest jobs added (sorted by date) |
| Fulltime Jobs | "fulltime" | All full-time positions |
| Parttime Jobs | "parttime" | All part-time positions |
| Freelance Jobs | "freelance" | All freelance/contract work |

### ✅ Jobs Page
- Search jobs by title
- View all available positions
- Click to see full details

## Context Functions Available

### In your components, use the `useJobs()` hook:

```javascript
const { 
  jobs,              // All jobs array
  filteredJobs,      // Currently filtered jobs
  loading,           // Loading state
  error,             // Error message if any
  getJobById,        // Get single job by ID
  getRecentJobs,     // Get recent jobs (returns 4 by default)
  getJobsByType,     // Get jobs by type (fulltime, parttime, etc.)
  filterJobsByType,  // Filter displayed jobs by type
  searchJobs,        // Search jobs by title
  addJob,            // Add new job (async)
  updateJob,         // Update existing job (async)
  deleteJob          // Delete job (async)
} = useJobs();
```

## Data Structure

Each job object contains:

```javascript
{
  id: "Firebase_generated_ID",           // Auto-generated by Firebase
  title: "Job Title",                    // e.g., "Electrician"
  type: "Fulltime",                      // Fulltime, Parttime, Freelance, etc.
  rate: "PHP 800-1500/m",                // Salary/rate
  companyName: "Company Name",           // Employer
  location: "City, Country",             // Job location
  vacancy: "5",                          // Number of positions
  hours: "40-50h/week",                  // Hours per week
  description: "Detailed description",   // Full job description
  workplace: "On-site",                  // On-site, Remote, Hybrid
  education: "Trade Certification",      // Education requirement
  experience: "3+ years",                // Experience needed
  skills: ["skill1", "skill2"],          // Required skills array
  icon: "⚡",                            // Emoji icon
  image: "/images/sourced/...",         // Job image URL
  createdAt: "2024-02-28T...",          // Creation timestamp
  updatedAt: "2024-02-28T..."           // Last update timestamp
}
```

## How It Works - User Flow

### User Journey:

1. **Homepage** 
   - Displays recent, fulltime, parttime, and freelance jobs
   - Click tabs to view different categories
   - Click job card to see details

2. **Jobs Page**
   - Search for specific jobs
   - See all available positions
   - Click to view full details

3. **Job Details Page**
   - View complete job information
   - Company details
   - Requirements and salary

### Admin Journey:

1. **Admin Panel** (`/admin`)
   - View all jobs in table
   - Click "Add New Job" to create
   - Click "Edit" to modify
   - Click "Delete" to remove
   
2. **Form Submission**
   - Fill required fields (title, company, location)
   - Submit form
   - See success/error message
   - Changes appear immediately on homepage

## Database Operations

### Add Job
```javascript
const newJob = {
  title: "Welder",
  type: "Fulltime",
  // ... other fields
};
await addJob(newJob);
```

### Update Job
```javascript
const updates = {
  rate: "PHP 1200-1600/m",
  vacancy: "3"
};
await updateJob(jobId, updates);
```

### Delete Job
```javascript
await deleteJob(jobId);
```

### Get Jobs by Type
```javascript
const fulltimeJobs = getJobsByType("fulltime", 4);
const freelanceJobs = getJobsByType("freelance", 4);
```

### Get Recent Jobs
```javascript
const recentJobs = getRecentJobs(4);  // Returns 4 most recent
```

## Common Issues & Solutions

### Jobs not showing on homepage?
- Check that jobs have correct `type` field (case-insensitive)
- Verify `createdAt` timestamp exists
- Clear browser cache and reload

### Edit button doesn't work?
- Make sure you have write permissions in Firebase
- Check browser console for errors
- Verify job ID is valid

### Search not working?
- Site searches by `title` field
- Ensure job title is not empty
- Search is case-insensitive

## Important Notes

⚠️ **Before Production:**
- Set up Firebase Authentication for admin protection
- Configure Database Security Rules
- Test all operations thoroughly
- Set up error monitoring

📝 **Job Type Values:**
These must match for filtering to work:
- "Fulltime"
- "Parttime"  
- "Freelance"
- "Contract"
- "Temporary"

🔒 **Data Security:**
- Implement authentication checks in Firebase rules
- Only admins should be able to modify jobs
- Users can read but not edit jobs
